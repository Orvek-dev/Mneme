# V1 Dogfood Evidence Triage MVP

## Summary

Phase 35 adds a deterministic decision loop over dogfood evidence bundles. The
goal is to know whether a bundle generated by `scripts/v1-dogfood.sh` is
complete enough to support manual v1 dogfood work.

## Requirements

- [REQ-DOGFOOD-TRIAGE-001][One Command] The eval harness shall expose
  `mneme-eval dogfood-summary <bundle-dir>`.
- [REQ-DOGFOOD-TRIAGE-002][Artifact Gate] The summary shall require dogfood
  validation, dogfood fake run, dogfood `mneme-v1` run, dogfood acceptance,
  v1 readiness, post-init doctor, context, quality, and validate artifacts.
- [REQ-DOGFOOD-TRIAGE-003][Decision Status] The summary shall return
  `ready_for_manual_dogfood` only when all required artifacts pass.
- [REQ-DOGFOOD-TRIAGE-004][Script Integration] `scripts/v1-dogfood.sh` shall
  write `dogfood-summary.json` into the evidence bundle.
- [REQ-DOGFOOD-TRIAGE-005][Automation] The release quality gate shall verify
  the dogfood summary command and generated bundle decision.
- [REQ-DOGFOOD-TRIAGE-006][Documentation] Public docs shall explain the triage
  command, required artifacts, and decision status.
